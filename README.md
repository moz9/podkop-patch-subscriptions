# Podkop Subscription URLTest Patch

Этот репозиторий не является форком Podkop и не содержит исходный код Podkop целиком.

Оригинальный проект: [itdoginfo/podkop](https://github.com/itdoginfo/podkop).

Здесь хранится только патч, который добавляет режим `Микс` для подписок и ручных proxy-ссылок, вкладку управления конфигами из подписок и безопасный автоподбор DNS.

## Установка и обновление на OpenWrt

Выполнять уже внутри SSH-сессии на роутере:

```sh
u=https://raw.githubusercontent.com/moz9/podkop-patch-subscriptions/main/i;f=/tmp/podkop-i;wget --no-check-certificate -O $f $u&&sh $f
```

Это одна универсальная команда для первой установки и последующих обновлений.

Что делает установщик:

- если Podkop не установлен, ставит официальный Podkop, затем применяет патч подписок;
- если установленная версия Podkop входит в список совместимости патча, сохраняет её и не запускает рискованное автоматическое обновление пакетов;
- официальный Podkop обновляется отдельно через центр обновлений или явно с `PODKOP_PATCH_FORCE_PODKOP_UPDATE=1`;
- перед обновлением сверяет последнюю версию Podkop со списком совместимости патча и не устанавливает неподдерживаемый релиз;
- если Podkop старее `0.7.0`, останавливается с понятной ошибкой: такие версии требуют ручной миграции официального Podkop;
- если патч `Микс` уже актуален, завершает работу с `PODKOP_PATCH_NOOP=1`;
- сохраняет настройки Podkop, подписки, кеш подписок и исключённые конфиги.

Если `raw.githubusercontent.com` недоступен, запасной вариант через CDN. После свежей публикации CDN может некоторое время отдавать предыдущую версию установщика:

```sh
u=https://fastly.jsdelivr.net/gh/moz9/podkop-patch-subscriptions@main/i;f=/tmp/podkop-i;wget --no-check-certificate -O $f $u&&sh $f
```

Если на роутере сломан DNS, но прямой HTTPS до GitHub доступен:

```sh
f=/tmp/podkop-i;wget --no-check-certificate --header='Host: raw.githubusercontent.com' -O $f https://185.199.108.133/moz9/podkop-patch-subscriptions/main/i&&sh $f
```

Этот вариант помогает скачать наш установщик без DNS. Для первой установки официального Podkop всё равно нужен рабочий доступ к GitHub/OpenWrt downloads.

Установщик:

- перед обновлением сохраняет `/etc/config/podkop` и `/etc/podkop`, чтобы не потерять настройки Podkop, кеш подписок и исключённые конфиги;
- безопасно пропускает автоматическое обновление официального Podkop для любой поддерживаемой установленной версии;
- скачивает runtime-патч, LuCI-файлы и русскую локализацию;
- делает резервную копию изменяемых файлов в `/root/podkop-patch-subscriptions-backup-*`;
- по умолчанию хранит только 2 последние резервные копии;
- применяет патч поверх актуального Podkop;
- проверяет shell-синтаксис;
- перезагружает Podkop и LuCI;
- при ошибке возвращает файлы из резервной копии.

## Что добавляет патч

- режим `Микс` для чтения proxy-конфигов из HTTP/HTTPS-подписок и ручного добавления отдельных proxy-ссылок;
- поддержку нескольких подписок в одной секции;
- кеш последней рабочей версии подписки;
- фильтрацию неподдерживаемых Podkop конфигов до генерации sing-box;
- отдельный список неподдерживаемых конфигов в дашборде;
- вкладку `Подписки` для включения и исключения отдельных конфигов;
- пакетное применение изменений с одним перезапуском Podkop;
- компактные действия во вкладке `Подписки`: обновить, ping, быстрый тест скорости, обновить патч;
- кеш community subnet списков, чтобы быстрый reload не оставлял `podkop_subnets` пустым;
- однокнопочный подбор полной связки `протокол + основной DNS + bootstrap DNS` по реальным запросам с роутера;
- единый глобальный рейтинг UDP, DoH и DoT: по умолчанию один тест последовательно проверяет все три режима и выбирает лучшую связку независимо от текущего режима Podkop;
- возможность оставить для теста один или два протокола, чтобы сократить время проверки;
- статистически устойчивый DNS-тест с прогревом, 8 bootstrap- и 20 основных измерений, медианой, P90 и межквартильным разбросом;
- проверка совместимости с Google Play, Google CDN, Steam, Microsoft, GitHub и CDN-сервисами; DNS с фильтрацией не выбирается как универсальный, пока доступен стабильный нефильтрующий вариант;
- отдельные проверки выбранных community-списков Podkop, включая YouTube, Telegram, Meta, Discord, Twitter и Google AI; недоступный выбранный сервис исключает DNS из надёжных рекомендаций;
- community-проверки используют отвечающие DNS-имена из актуальных списков allow-domains, чтобы не отклонять все DNS из-за неактивного домена или корня CDN без A-записи;
- выбор основных и bootstrap DNS для проверки, чтобы не запускать полный каталог при каждом тесте;
- сохраняемые пользовательские DNS-кандидаты отдельно для UDP, DoH и DoT;
- автоматическая рекомендация только из известных публичных нефильтрующих пар; текущие, пользовательские и WAN DNS остаются в таблице только для сравнения и явной ручной установки;
- один понятный итог подбора: отдельные карточки основной и независимой резервной связок плюс одна кнопка безопасной установки обеих;
- подробная таблица по умолчанию свёрнута и остаётся экспертным режимом; сравниваемые, но неподходящие для рекомендации строки установить нельзя;
- сохранение последнего завершённого теста под раскрываемым блоком даже после установки, отката и повторного открытия страницы;
- автоматический выбор двух независимых полных DNS-пар: предпочтительной и резервной, по возможности с разными основными и bootstrap-провайдерами;
- лёгкий watchdog без дополнительного DNS-сервиса: после двух последовательных отказов активной пары он проверяет вторую, переключает весь DNS-тракт Podkop, проверяет sing-box и FakeIP и автоматически откатывает неудачное переключение;
- отдельные ручные поля протокола, основного и bootstrap DNS для резервной пары; при ручном изменении предпочтительной пары штатная кнопка LuCI снова активирует её;
- сохранение текущей DNS-пары, когда несколько вариантов статистически равны, чтобы повторные проверки не вызывали случайные переключения;
- применение рекомендованных DNS с повторной проверкой прикладных доменов через sing-box и dnsmasq, проверкой FakeIP, автоматическим откатом при ошибке и отдельной кнопкой ручного отката;
- обновление DNS-полей после применения без перезагрузки страницы и без повторного запуска завершённой операции;
- синхронизация внутреннего UCI-состояния LuCI после фонового применения, чтобы последующий ручной выбор DNS снова штатно сохранялся кнопкой применения;
- компактный центр обновлений в диагностике: проверка Podkop и патча, безопасное совместное обновление, блокировка неподдерживаемых релизов, просмотр и копирование терминального лога;
- русские строки интерфейса для добавленных элементов.

Совместимость публикуется в `openwrt/update-manifest.json`. Новая официальная версия Podkop становится доступна через кнопку обновления только после добавления её в `supportedPodkopVersions` и проверки патча на этой версии.

## Патч для исходников

Если нужно применить изменения к локальному checkout Podkop:

```sh
git clone https://github.com/itdoginfo/podkop.git
git clone https://github.com/moz9/podkop-patch-subscriptions.git
cd podkop
git am ../podkop-patch-subscriptions/patches/*.patch
```

Runtime-установка для OpenWrt использует отдельные файлы из `openwrt/`.
